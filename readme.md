# 千恋万花 立绘合成工具

Python GUI 工具，用于可视化预览和导出游戏《千恋万花》的角色立绘，替代手动 Photoshop 合成工作流。

## 依赖

- Python 3
- Pillow（PIL）

```bash
pip install Pillow
```

## 运行

```bash
python sprite_composer.py
```

或双击 `start.bat`。

## 项目结构

```
├── config.ini                  # 角色配置（用户可编辑）
├── sprite_composer.py          # 主程序
├── start.bat                   # Windows 启动脚本
├── bug.md                      # Bug 记录与设计改进
├── 立绘合成工具设计总结.md      # 详细设计文档
├── readme.md                   # 本文件
└── 所有角色资源/
    ├── 芳乃/
    │   ├── 芳乃.stand          # 变体入口（含 a/b 视图定义）
    │   ├── 芳乃a.txt           # 高分辨率图层布局
    │   ├── 芳乃a_0.txt         # 低分辨率图层布局
    │   ├── 芳乃a_info.txt      # 资源索引（服装/表情映射）
    │   ├── 芳乃a_{id}.png      # 高分辨率图层图片
    │   └── 芳乃a_0_{id}.png    # 低分辨率图层图片
    ├── 茉子/
    ├── レナ/
    └── ...（共 12 个角色）
```

## 配置文件（config.ini）

```ini
[角色名]
stand = ./所有角色资源/{角色名}/{角色名}.stand
decorations = 显示名: 匹配关键词[, bind: 服装关键词]; ...
```

- `stand`：指向角色的 `.stand` 文件，其余文件自动发现
- `decorations`（可选）：分号分隔的装饰物条目。`bind:` 前缀表示该装饰物已内置在对应服装资源中，不创建独立 CheckBox

### 装饰物配置格式

| 格式 | 说明 | 示例 |
|------|------|------|
| `显示名: 关键词` | 独立 CheckBox，按关键词子串匹配图层 | `涙HL（泪水高光）: 涙HL` |
| `显示名: 关键词, bind: 服装关键词` | 不创建 CheckBox，随匹配服装自动激活 | `ケモミミ（兽耳）: ケモミミ, bind: 耳` |

## 数据文件格式

### 文件编码

所有 `.txt` 和 `.stand` 文件均为 **UTF-16 LE with BOM**（`FF FE`）。

### .stand 文件

自定义 JSON-like 格式，定义角色的所有视图变体：

```json
[
  { "filename": "芳乃a", "xoffset": 32, "yoffset": 180, "leveloffset": [...] },
  { "filename": "芳乃b", "xoffset": 42, "yoffset": 180, "leveloffset": [...] }
]
```

### _info.txt 文件

Tab 分隔，定义服装和表情的资源映射：

```
dress	巫女服	diff	1	巫女服
dress	巫女服	diff	1	髪かぶせ
dress	巫女服	diff	2	巫女服腕差分
face	01	base	表情/ベース
face	01h	base	表情/ベース
face	01h	base	頬
face	03n	base	涙あり/203 涙
```

- **dress**：`dress \t <服装名> \t diff \t <序号> \t <资源名>`
- **face**：`face \t <编号> \t base \t <资源路径>`
- 资源路径中 `/` 前为所属组名，`/` 后为图层名
- 表情后缀：`h`=红晕(頬)、`n`=泪水、`r`=去高光、`d`=暗化效果

### 图层布局 .txt 文件

Tab 分隔，每行定义一个图层或组容器：

| 字段 | 含义 | 说明 |
|------|------|------|
| layer_type | 图层类型 | 0=图片层, 2=组容器 |
| name | 资源名 | 与 _info.txt 中资源名匹配 |
| left/top | 左上坐标 | 像素 |
| width/height | 图层尺寸 | 像素 |
| image_type | 图片类型 | 13=普通Alpha混合, 16=正片叠底(暗化) |
| opacity | 不透明度 | 0-255 |
| visible | 默认可见性 | 0/1 |
| layer_id | 唯一标识 | 对应 PNG 文件名 `{prefix}_{id}.png` |
| group_layer_id | 所属组 | 指向 layer_type=2 的组容器 |

## 使用说明

### 主界面

启动后显示角色列表，双击或选中后点击「打开选中角色」进入编辑界面。

### 角色编辑界面

| 选择项 | 说明 | 多选/空选 |
|--------|------|-----------|
| 站位变体 | 角色的不同视图（a/b） | 允许空选 |
| 分辨率 | 高清（3600×5100）/ 低清（1800×2550） | 不允许空选 |
| 服装 | 从 _info.txt 解析的服装列表 | 允许空选 |
| 表情 | 含 h/n/r/d 后缀变体 | 允许空选 |
| 装饰物 | 数据驱动的独立 CheckBox | 多选叠加 |

### 预览区

- 棋盘格背景表示透明区域
- 鼠标滚轮缩放（10%~400%）
- 拖拽平移
- 点击倍率标签可手动输入数值（1~400）
- 「适应窗口」按钮自动适配画布

### 导出

点击「导出 PNG」保存当前组合为 RGBA PNG，画布自动裁剪至可见图层包围盒。

## 核心设计

- **数据驱动**：装饰物通过 `config.ini` 配置，不硬编码角色名或图层名
- **组感知匹配**：表情资源的路径前缀（如 `表情/ベース`）限定图层匹配范围，防止同名冲突
- **表情后缀自动推导**：`n` 后缀自动加载基础表情、`d` 后缀图层使用正片叠底混合
- **公共资源**：所有服装共有的资源（如 `髪かぶせ`）仅在选择服装时激活
- **所见即所得**：预览底图为可见图层包围盒，无名义尺寸空白
